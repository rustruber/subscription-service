# Changelog

## [0.3.2] - 2026-06-22
### Fixed
1. Логика расчета суммарной стоимости (Total Cost) реализована на уровне БД, но не учитывает количество месяцев действия подписки внутри периода (SQL-запрос просто суммирует price, где `start_date` попадает в диапазон, без корректировки на длительность, что может быть математически неточно для частичных месяцев).
2. Эндпоинт Update не использует `gorilla/mux` vars для ID, хотя использует Put, что корректно, но обработка `EndDate` в handler (проверка на пустую строку перед парсингом) может быть избыточной при наличии указателя.
3. В List реализована пагинация (limit/offset), что хорошо.
4. В main.go есть "хвост" кода (функция `foo`), создающая нагрузку и утечку памяти, которая запускается в горутине (`go foo()`). Это подозрительный код, не относящийся к логике сервера, который может привести к падению сервиса в проде.
5. В `runMigrations` дублируется логика создания таблицы и индексов, которая уже есть в миграциях. В случае синхронизации это может привести к ошибкам или рассинхрону состояний. Лучше полагаться только на migrate контейнер.
6. Использование стандартного `log` вместо структурированных логеров типа `zap` или `zerolog` упрощает отладку.
7. Отсутствует graceful shutdown для HTTP сервера (сигналы SIGTERM/SIGINT не обрабатываются, `http.ListenAndServe` блокирует главную горутину).
8. В `internal/adapter/repository/postgres/repository.go` функция `GetTotalCost` использует ручную конкатенацию строки запроса `fmt.Sprintf` для плейсхолдеров (`$3`, `$4`). Несмотря на то, что данные передаются безопасно (через `args`), стиль "в лоб" с конкатенацией считается плохим тоном (лучше использовать библиотеку запросов или `sq`).
